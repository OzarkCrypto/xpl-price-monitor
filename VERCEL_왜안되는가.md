# Vercel이 왜 안 되는가?

## 🔍 근본 원인

Vercel은 **서버리스 플랫폼**이므로 Flask WSGI 앱을 직접 실행하는 방식이 아닙니다.

### 문제점

1. **서버리스 함수 형식**
   - Vercel은 각 요청마다 함수를 실행
   - Flask 앱의 `app.run()`은 실행되지 않음
   - `if __name__ == '__main__'` 블록이 실행되지 않음

2. **백그라운드 스레드**
   - 서버리스 환경에서는 백그라운드 스레드가 작동하지 않음
   - 실시간 모니터링 불가능

3. **상태 유지**
   - 요청 간에 메모리 상태가 유지되지 않음
   - 히스토리 데이터 손실

4. **템플릿 경로**
   - Vercel Lambda 환경에서 경로 찾기 어려움
   - 절대 경로 vs 상대 경로 문제

5. **의존성 문제**
   - 일부 패키지가 서버리스 환경에서 작동하지 않을 수 있음
   - aiohttp, asyncio 등 비동기 라이브러리

## 💡 해결책: Railway 사용

**Railway는 이 프로젝트에 완벽하게 맞습니다:**

### Railway의 장점
- ✅ Flask 앱 직접 지원 (WSGI)
- ✅ 백그라운드 스레드 지원
- ✅ 상태 유지 가능
- ✅ 슬립 모드 없음
- ✅ 설정 파일 자동 인식
