# Vercel 크래시 문제 최종 해결

## 🔍 문제 원인

Vercel 서버리스 함수가 계속 크래시되는 이유:

1. **Flask 앱 구조**: Vercel은 Flask를 직접 지원하지 않음
2. **템플릿 경로**: Vercel 환경에서 템플릿 경로를 찾지 못함
3. **백그라운드 스레드**: 서버리스 환경에서 스레드가 작동하지 않음
4. **의존성 문제**: 일부 패키지가 Vercel 환경에서 작동하지 않을 수 있음

## ✅ 해결 방법

### 수정된 내용

1. **템플릿 경로 수정**: Vercel 환경에서도 템플릿을 찾을 수 있도록 수정
2. **에러 핸들링 강화**: import 실패 시 에러 메시지 표시
3. **환경 변수 설정**: Vercel 환경 감지 및 백그라운드 스레드 비활성화

## 🚀 재배포

코드 수정 후 GitHub에 푸시하면 Vercel이 자동으로 재배포합니다.

```bash
git push origin main
```

## ⚠️ Vercel의 근본적인 제한사항

**Vercel은 이 프로젝트에 적합하지 않을 수 있습니다:**

1. **서버리스 모델**: 요청이 있을 때만 실행
2. **상태 유지 불가**: 메모리 상태가 요청 간에 유지되지 않음
3. **백그라운드 작업 불가**: 실시간 모니터링 어려움
4. **실행 시간 제한**: 무료 플랜 10초 제한

## 💡 권장 해결책: Railway 사용

**Railway가 이 프로젝트에 가장 적합합니다:**

### Railway 배포 (5분)

1. **https://railway.app** 접속
2. **"Start a New Project"** 클릭
3. **"Deploy from GitHub repo"** 선택
4. 저장소 선택: `OzarkCrypto/xpl-price-monitor`
5. **자동 배포!** (설정 파일 자동 인식)

### Railway의 장점

- ✅ 백그라운드 스레드 지원
- ✅ 상태 유지 가능
- ✅ 슬립 모드 없음
- ✅ 설정 파일 자동 인식 (`railway.json`)
- ✅ 무료 크레딧 제공
